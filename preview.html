About this preview

The widget now talks to its host via the official @modelcontextprotocol/ext-apps SDK. This preview acts as a minimal mock host: it answers ui/initialize, replies to tools/call JSON-RPC requests (server path), and pushes ui/notifications/tool-result + ui/notifications/interpretation notifications (LLM path). All messages flow over postMessage exactly as they would from a real MCP Apps host.

Simulate the LLM reasoning path

In a real MCP host (Claude Desktop, ChatGPT Apps), typing a math question in chat causes the LLM to call a primitive tool, and the host pushes the result to the widget via ui/notifications/tool-result. These buttons simulate that push so you can see the third path light up in the widget without running the full host.

V2 — scientific primitives

The widget itself now exposes √x, x², xy, log₁₀, and ln keys that route directly to these primitives via app.callServerTool (badge: server). The buttons below simulate the alternative path — the host LLM pushing the same tool results in via ui/notifications/tool-result (badge: llm).

V3 — natural-language word problems

Simulates the full V3 teaching loop: user phrasing → interpreted math → executed primitive tool calls → final answer. The host pushes a ui/notifications/interpretation envelope first (phrasing, concept, expression), then issues the primitive tool calls, then pushes a completion envelope with the final answer. The server stays a flat collection of primitives — interpretation lives in the LLM and is *displayed* by the widget.
